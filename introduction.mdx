Kill Bill is the leading open-source subscription billing and payments platform. Founded in 2010, it has been adopted by SaaS companies and e-commerce businesses that need complete control over their billing infrastructure. You get automated invoicing, multi-gateway payment processing, usage metering, dunning, and multi-tenancy — all accessible through a REST API and backed by a persistent event bus.

Because Kill Bill is open-source under the Apache 2.0 license, you run it on your own infrastructure. You own the data, you control the configuration, and you extend the platform through plugins without waiting on a SaaS vendor's roadmap.

## Architecture overview

Kill Bill is composed of a set of internal modules that interact through a persistent event bus. Understanding how they fit together helps you reason about billing flows.

```
Account → Catalog → Subscription → Invoice → Payment
```

| Module | Responsibility |
|--------|---------------|
| **Account** | Stores customer identity, currency, time zone, and payment method references. |
| **Catalog** | Versioned XML definition of products, plans, phases, and pricing rules. Uploaded per tenant. |
| **Entitlement / Subscription** | Manages the lifecycle of a customer's subscription to a plan — creation, changes, cancellation, and blocking states. |
| **Invoice** | Listens for subscription events and generates invoices automatically. Handles proration, credits, and adjustments. |
| **Payment** | Processes invoices against a payment method via gateway plugins. Tracks transaction state and handles retries. |
| **Overdue** | Evaluates unpaid invoice age against configured thresholds and transitions accounts through overdue states. |
| **Tenant** | Isolates catalog, configuration, and data per logical tenant within a single Kill Bill deployment. |

When a subscription is created or changes phase, the entitlement module publishes an event. The invoice module consumes that event and generates an invoice. Once an invoice exists, the payment module attempts to collect using the account's default payment method.

<Note>
  Every billing action in Kill Bill is audit-logged. You can retrieve the full history of any object — account, subscription, invoice, or payment — including the context of who made each change and why.
</Note>
