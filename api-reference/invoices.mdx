---
title: "Invoices API — billing and invoice management"
description: "Retrieve, generate, commit, void, and pay Kill Bill invoices; add item adjustments and external charges; manage invoice HTML templates and translations."
---

Kill Bill generates invoices automatically at the end of each billing period. An invoice aggregates all charges due for a given target date across all of a customer's active subscriptions. You can also create invoices manually for external charges, migration items, or dry-run previews.

Each invoice line item has an `itemType` that indicates its origin:

| Item type | Description |
|---|---|
| `RECURRING` | A recurring charge from a subscription plan phase. |
| `FIXED` | A one-time charge from a plan phase (e.g., setup fee). |
| `USAGE` | A usage-based charge calculated from recorded usage. |
| `EXTERNAL_CHARGE` | A charge added manually via the external charges API. |
| `CBA_ADJ` | A credit balance adjustment applied to reduce the invoice balance. |
| `ITEM_ADJ` | An item-level adjustment that reduces or cancels a prior charge. |
| `TAX` | A tax item generated by a tax plugin. |
| `PARENT_SUMMARY` | A summary item on a parent account invoice representing a child account's charges. |

## Get invoice by UUID

<br />

`GET /1.0/kb/invoices/{invoiceId}`

Retrieves a single invoice by its Kill Bill UUID, including all line items.

### Path parameters

<ParamField path="invoiceId" type="string" required>
  The Kill Bill UUID of the invoice.
</ParamField>

### Query parameters

<ParamField query="withChildrenItems" type="boolean" default="false">
  When `true` and this is a parent invoice, includes the line items from all child account invoices inline.
</ParamField>

<ParamField query="audit" type="string" default="NONE">
  Audit log detail level. One of `NONE`, `MINIMAL`, or `FULL`.
</ParamField>

### Response

Returns an [Invoice object](#invoice-object).

```bash Example request
curl -v \
  -u admin:password \
  -H "X-Killbill-ApiKey: bob" \
  -H "X-Killbill-ApiSecret: lazar" \
  "http://localhost:8080/1.0/kb/invoices/2ad52f53-85ae-408a-9879-32a7e59dd03d"
```

## Trigger invoice generation

<br />

`POST /1.0/kb/invoices`

Manually triggers invoice generation for an account up to a given target date. Kill Bill generates any charges that are due on or before that date and have not yet been invoiced.

Returns `404 Not Found` when there is nothing to invoice (no charges due before the target date).

### Query parameters

<ParamField query="accountId" type="string" required>
  UUID of the account for which to generate an invoice.
</ParamField>

<ParamField query="targetDate" type="string">
  ISO 8601 date (`YYYY-MM-DD`) through which to generate charges. Defaults to today.
</ParamField>

### Response

`201 Created` — Returns a `Location` header pointing to the new invoice.

```bash Example request
curl -v \
  -u admin:password \
  -H "X-Killbill-ApiKey: bob" \
  -H "X-Killbill-ApiSecret: lazar" \
  -H "Content-Type: application/json" \
  -H "X-Killbill-CreatedBy: demo" \
  -X POST \
  "http://localhost:8080/1.0/kb/invoices?accountId=2ad52f53-85ae-408a-9879-32a7e59dd03d&targetDate=2025-02-01"
```

## Generate dry-run invoice

<br />

`POST /1.0/kb/invoices/dryRun`

Generates a preview invoice without persisting it. Use this to show customers what they will be charged before a billing period closes or before a plan change takes effect.

Returns `204 No Content` when there would be nothing to invoice.

### Query parameters

<ParamField query="accountId" type="string" required>
  UUID of the account.
</ParamField>

<ParamField query="targetDate" type="string">
  Target date for the dry-run invoice. Required unless `dryRunType` is `UPCOMING_INVOICE`.
</ParamField>

### Request body

An optional `InvoiceDryRunJson` object. When the body is omitted or empty, Kill Bill previews charges through `targetDate`.

<ParamField body="dryRunType" type="string">
  Type of dry run.

  | Value | Description |
  |---|---|
  | `UPCOMING_INVOICE` | Preview the next invoice that would be generated. |
  | `SUBSCRIPTION_ACTION` | Preview the invoice impact of a specific subscription action. |
  | `TARGET_DATE` | Preview charges through the given `targetDate`. |
</ParamField>

<ParamField body="dryRunAction" type="string">
  The subscription action to preview. One of `START_BILLING`, `CHANGE`, or `STOP_BILLING`. Required when `dryRunType` is `SUBSCRIPTION_ACTION`.
</ParamField>

<ParamField body="subscriptionId" type="string">
  UUID of the subscription affected by the action. Required for `CHANGE` and `STOP_BILLING` actions.
</ParamField>

<ParamField body="bundleId" type="string">
  UUID of the bundle. Required for `CHANGE` and `STOP_BILLING` actions.
</ParamField>

<ParamField body="planName" type="string">
  Target plan for a `START_BILLING` or `CHANGE` action.
</ParamField>

<ParamField body="billingPolicy" type="string">
  Billing policy to apply to the simulated action.
</ParamField>

### Response

`200 OK` — Returns a preview [Invoice object](#invoice-object) (not stored). `204 No Content` if nothing would be invoiced.

## Commit draft invoice

<br />

`PUT /1.0/kb/invoices/{invoiceId}`

Commits a draft invoice, making it final and triggering automatic payment processing. Invoices in `DRAFT` status are not automatically paid.

### Path parameters

<ParamField path="invoiceId" type="string" required>
  UUID of the draft invoice to commit.
</ParamField>

### Response

`204 No Content`

## Void invoice

<br />

`DELETE /1.0/kb/invoices/{invoiceId}`

Voids an invoice, marking it as uncollectable. The invoice remains in the system for audit purposes but its balance is zeroed. Any associated `CBA_ADJ` items are removed.

<Warning>
  Voiding an invoice that has already been paid will not automatically refund the payment. You must issue a refund separately.
</Warning>

### Path parameters

<ParamField path="invoiceId" type="string" required>
  UUID of the invoice to void.
</ParamField>

### Response

`204 No Content`

## Add item adjustment

<br />

`POST /1.0/kb/invoices/{invoiceId}`

Adds an `ITEM_ADJ` item that reduces or fully credits a specific existing line item on an invoice.

### Path parameters

<ParamField path="invoiceId" type="string" required>
  UUID of the invoice containing the item to adjust.
</ParamField>

### Query parameters

<ParamField query="requestedDate" type="string">
  ISO 8601 date for the adjustment. Defaults to today.
</ParamField>

### Request body

<ParamField body="accountId" type="string" required>
  UUID of the account that owns the invoice.
</ParamField>

<ParamField body="invoiceItemId" type="string" required>
  UUID of the invoice line item to adjust.
</ParamField>

<ParamField body="amount" type="number">
  Amount of the adjustment. Must be positive and no greater than the original item amount. If omitted, the full item amount is adjusted (complete credit).
</ParamField>

<ParamField body="currency" type="string">
  ISO 4217 currency code. Must match the invoice currency. Required when `amount` is specified.
</ParamField>

<ParamField body="description" type="string">
  Human-readable description of the adjustment reason.
</ParamField>

### Response

`201 Created` — Returns a `Location` header pointing to the updated invoice.

```bash Example request
curl -v \
  -u admin:password \
  -H "X-Killbill-ApiKey: bob" \
  -H "X-Killbill-ApiSecret: lazar" \
  -H "Content-Type: application/json" \
  -H "X-Killbill-CreatedBy: demo" \
  -X POST \
  --data-binary '{
    "accountId": "2ad52f53-85ae-408a-9879-32a7e59dd03d",
    "invoiceItemId": "b7f9a12d-3c2e-4f1a-8b9c-5d6e7f8a9b0c",
    "amount": 10.00,
    "currency": "USD",
    "description": "Goodwill credit for service disruption"
  }' \
  "http://localhost:8080/1.0/kb/invoices/3ef8b23e-4d3f-5g2b-9c0d-6e7f8a9b0c1d"
```

## Add external charge (to existing invoice)

<br />

`POST /1.0/kb/invoices/{invoiceId}/charges/{accountId}`

Adds an `EXTERNAL_CHARGE` item to an existing invoice. Use this to add charges that originate outside of Kill Bill's subscription engine (e.g., professional services, one-time setup fees).

### Path parameters

<ParamField path="invoiceId" type="string" required>
  UUID of the invoice to add the charge to.
</ParamField>

<ParamField path="accountId" type="string" required>
  UUID of the account.
</ParamField>

### Query parameters

<ParamField query="requestedDate" type="string">
  Effective date of the charge. Defaults to today in the account's time zone.
</ParamField>

<ParamField query="autoCommit" type="boolean" default="false">
  When `true`, immediately commits the invoice if it is currently in `DRAFT` status.
</ParamField>

### Request body

An array of invoice item objects.

<ParamField body="[].amount" type="number" required>
  Charge amount.
</ParamField>

<ParamField body="[].currency" type="string">
  ISO 4217 currency. Defaults to the account's currency.
</ParamField>

<ParamField body="[].description" type="string">
  Human-readable description of the charge.
</ParamField>

<ParamField body="[].bundleId" type="string">
  Associate the charge with a specific bundle.
</ParamField>

<ParamField body="[].subscriptionId" type="string">
  Associate the charge with a specific subscription.
</ParamField>

<ParamField body="[].startDate" type="string">
  Start date for the charge period (`YYYY-MM-DD`).
</ParamField>

<ParamField body="[].endDate" type="string">
  End date for the charge period (`YYYY-MM-DD`).
</ParamField>

### Response

Returns an array of created invoice item objects.

## Add external charge (auto-create invoice)

<br />

`POST /1.0/kb/invoices/charges/{accountId}`

Creates an `EXTERNAL_CHARGE` item. If no existing open invoice is found for the target date, Kill Bill automatically creates a new invoice to hold the charge.

### Path parameters

<ParamField path="accountId" type="string" required>
  UUID of the account to charge.
</ParamField>

### Query parameters

<ParamField query="requestedDate" type="string">
  Effective date of the charge.
</ParamField>

<ParamField query="autoCommit" type="boolean" default="false">
  Immediately commit the invoice after adding the charge.
</ParamField>

### Request body

Same as [add external charge to existing invoice](#add-external-charge-to-existing-invoice).

### Response

Returns an array of created invoice item objects.

```bash Example request
curl -v \
  -u admin:password \
  -H "X-Killbill-ApiKey: bob" \
  -H "X-Killbill-ApiSecret: lazar" \
  -H "Content-Type: application/json" \
  -H "X-Killbill-CreatedBy: demo" \
  -X POST \
  --data-binary '[
    {
      "amount": 250.00,
      "currency": "USD",
      "description": "Professional services — January 2025"
    }
  ]' \
  "http://localhost:8080/1.0/kb/invoices/charges/2ad52f53-85ae-408a-9879-32a7e59dd03d?autoCommit=true"
```

## Pay specific invoice

<br />

`POST /1.0/kb/invoices/{invoiceId}/payments`

Triggers a payment attempt for a single invoice using the account's default payment method or a specified one.

### Path parameters

<ParamField path="invoiceId" type="string" required>
  UUID of the invoice to pay.
</ParamField>

### Query parameters

<ParamField query="externalPayment" type="boolean" default="false">
  When `true`, records the payment as an external payment without charging a payment plugin. Cannot be combined with `paymentMethodId`.
</ParamField>

### Request body

<ParamField body="accountId" type="string" required>
  UUID of the account.
</ParamField>

<ParamField body="purchasedAmount" type="number">
  Amount to collect. Defaults to the full invoice balance.
</ParamField>

<ParamField body="paymentMethodId" type="string">
  UUID of the payment method to use. Defaults to the account's default payment method.
</ParamField>

<ParamField body="paymentExternalKey" type="string">
  Caller-assigned external key for the payment transaction.
</ParamField>

### Response

`201 Created` — Returns a `Location` header pointing to the payment.

## Invoice HTML template

Manage the Mustache/HTML template used to render invoices for all accounts in the tenant.

### Get template

`GET /1.0/kb/invoices/template`

Returns the current HTML invoice template as `text/html`.

### Upload template

`POST /1.0/kb/invoices/template`

Uploads a new HTML invoice template. The request body is the raw HTML/Mustache template as `text/html`.

<ParamField query="deleteIfExists" type="boolean" default="false">
  Replace any existing template rather than failing if one already exists.
</ParamField>
